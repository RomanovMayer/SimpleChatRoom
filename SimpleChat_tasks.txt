Первая задача - минимальный функционал:
1) Общение клиентов через различные комнаты
	Что для этого необходимо:
		- Реализовать протокол прикладного уровня "клиент - сервер" и "сервер - клиент"
			для обмена сообщениями. Этот протокол должен быть легко расширяемым для 
			учитывания будующего функционала.
		- Релизовать передачу данных по сети.
			Подумать по поводу интерфейса - уверен в будущем будем часто переписывать
			реализацию, но интерфейс не должен при этом меняться!!!
		- Реализовать внутреннюю логику сервера.
			Клиент подключается к серверу, заходит в комнату, общается, выходит из комнаты.
			Все эти действия осуществляются с согласия Сервера.
		- Реализовать внутреннюю логику клиента
			Создать enum текущего состояния клиента. (Ожидание ответа от Сервера, Запрос к Серверу, ???)
			для грамотного взаимодействия с GUI.
			Логика клиентской стороны должны быть минимальной. Большую часть времени Клиентская сторона
			программы должна ожидать действий пользователя или сообщений от сервера. 
		- Реализовать GUI клиента.
			Выбор комнат, Просмотр участников комнаты, Набор сообщений.
